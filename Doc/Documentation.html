1. Beschreibung des Anwendungsszenarios

Die vorliegende Arbeit behandelt die Erstellung einer digitalen Bibliothek für eine private Schule in Dortmund. Hiermit wird es primär den Lehrenden und Lernenden ermöglicht, im Rahmen einer digitalen Bildung auf digitale Literaturen zuzugreifen. Ferner kann eine Liste für nicht vorhandene Literaturen erstellt werden.

2. Fachobjekte der Anwendung

In dieser Anwendung ist nur ein einziges Fachobjekt vorzufinden. Das hier ausgewählte Fachobjekt ist ein Buch. Es wurden vier Instanzen erzeugt, die in der Detailseite anzutreffen sind. Zum Buch-Objekt sind folgende Attribute angelegt worden : "titel, autor, herausgeber, erscheinungsjahr, umfang, isbn, status, sprache, beschreibung". Da lediglich ein einziges Fachobjekt vorhanden ist gibt es deswegen keine Assoziation.

3. Beschreibung und Begründung der Implementierungsentscheidungen

Der Client besteht aus vier Hauptseiten :
  • eine Listenseite : Sie listet die Bücher der bibliothek auf
  • eine Detailseite : Sie gibt sämtliche Informationen über eine Buchinstanz an
  • eine BuchAnlegen-Seite: die die Hinzufügung neuer Buchinstanzen ermöglicht
  • startseite: Startpunkt der Anwendung
  • Failure : Diese Seite wird bei nicht zutreffender Anfrage angezeigt.
  • eine Navigation zwischen den HTML-Seiten ist über Hyperlinks im Navigationsbereich möglich
  • Der Einsatz von "DIV" bezweckt die Definition eines Bereiches, um gewisse Elemente zusammen zu halten.
Das Layout wurde mit CSS generiert :
Um ein effizientes und einfaches Layout zu gewährleisten wurden die Layout-Mechanismen Flex und Grid eingesetzt. Beispielweise Body als Grid, um seine Kinder besser orientieren zu können. Des Weiteren wurden dessen Kinder als Flexbox definiert. Auf der Seite mit detaillierten Informationen wurde float zur Positionierung von Elementen angewendet.
Responsivität :
Die Definition eines Breackpoints bei einer Anzeigebreite von 480px soll das grundsätzliche Layout in ein anderes überführen. Ziel hierbei soll die Unterstützung kleiner mobiler Geräte sein.
Die Dynamik wurde mit Javascript erreicht :
Mit Javascript wurde unsere Anwendung um weitere Technologien bereichert. An dieser Stelle wurde der Fokus auf Benutzerinteraktionen gelegt und Mechanismen etabliert, die Ereignisse behandeln. Zum Beispiel ist es möglich, aud Wunsch, d.h. falls eine Literatur leider nicht vorhanden ist, diese in eine Liste einzutragen. Ebenfalls die Anzeige von den einzelnen Seiten (s.o.) wurde dynamisch realisiert. Darüber hinaus lassen sich drei Buchinstanzen dynamisch aus der Datenbank (Array) in die Detailseite anzeigen. Zu guter letzt werden die beim Ausfüllen des Formulars zum Anlegen von Büchern eingebenen Daten in die Detailseite zu einem Buch zusammengefasst. N.B: Es werden nur drei von den einzugebenden erforderlichen Daten bearbeitet.
Server :
Nachdem die Ressourcen definiert wurden mussten nun diese für die Bibliotheknutzer zugreibar gemacht werden. Hierzu wurden zuerst das Modul Node.js verwendet, das einen Server definiert und einige technische Methoden zur Verfügung stellt. Zusätzlich wurde mit Express Routing-Probleme gelöst. Der Einsatz von EJS machte es möglich, unsere ressourcen unkompliziert auszuliefern.

Damit auf diese Ressourcen zugegriffen werden kann werden folgende Anfragen unterstützt :

URL HTTP-Methode Aktion
/ GET Anzeige der Startseite
/Detailseite GET Anzeige der Detailseite
/Listenseite GET Anzeige der Listenseite
/Buchanlegen GET Anzeige der Buchanlage-seite
/Buchanlegen POST Hinzufügen eines neuen Buches

4. Start der Anwendung

Die Anwendung ist unter der URL : http://localhost:4000 erreichbar. Um die Anwendung starten zu können muss der Server in der bibapp.js Datei in Betrieb genommen werden.
